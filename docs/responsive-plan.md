# レスポンシブ対応プラン

携帯端末から快適に読めるようにするための段階計画です。各 Phase ごとにプレビュー確認 → フィードバックで次へ進みます。

## Phase 1 — レイアウト破綻の除去（優先）

- infobox / wikitable / コードブロックの横スクロール化
- 記事ページのサイドバーを本文下へ移動（グリッド解除時の順序調整）
- パディング・フォントサイズを小画面向けに縮小
- 画像（`.embed-image` 等）が突き抜けないように `max-width: 100%`

## Phase 2 — ナビゲーションの使いやすさ

- ヘッダーの検索・ナビをコンパクト化（折り返しや hamburger 的な集約）
- 目次の折り畳み動作が小画面で操作しやすいか確認
- タッチ領域の下限（44px 目安）

## Phase 3 — 読み心地の微調整

- 本文の行幅・行間、見出しの下マージン
- タグ・履歴リストの詰め方
- リンクの間隔（誤タップ防止）

## ブレークポイント（現状）

- `max-width: 960px` — 2カラムを1カラムへ
- `max-width: 640px` — page 幅調整・検索フォーム1列化

必要に応じて `480px` あたりを追加検討。
