**ローカルで記事プレビューを確認する手順**は、本番（GitHub Pages）に公開する前に、自分のPC上で記事の見た目を確認するための手順です。

> [!warning] 中級者向け以上の手順です
> この記事では '''Node.js のインストール''' と '''コマンドラインの使用''' が必要です。Git/GitHub Desktopの基本操作までしか触れたことのない方は[[はじめての記事投稿ガイド]]の手順だけで十分プルリクエストを出せます。レビュー時に管理者が気づいた問題はコメントで指摘されますし、軽微な記事は事前確認なしで問題ありません。<br>
> 「表示崩れの試行錯誤を自分で繰り返したい」「複数記事を一気に投稿する前にチェックしたい」「自分でサイト基盤側の修正に踏み込みたい」といった場合に活用してください。

== 何ができるようになるか ==

* PRを出す前に「この書き方で意図通り表示されるか」を試せる
* 表示崩れに気づいたら原因を切り分けて報告しやすい
* 大量の記事を投稿する前のセルフレビューに使える
* サイト基盤の修正を試す土台になる

== 必要なもの ==

* '''Node.js 20以降'''（後述の「[[#Node.jsをインストールする]]」参照）
* このリポジトリをクローン済みであること（[[はじめての記事投稿ガイド]]の「リポジトリをクローンする」参照）
* コマンドライン（コマンドプロンプト／PowerShell／ターミナル）に最低限触れたことがあること

== Node.jsをインストールする ==

# ブラウザで '''https://nodejs.org/''' にアクセスします
# '''「LTS」''' 版のインストーラーをダウンロードします
# インストーラーを実行し、デフォルト設定のまま進めます
# インストール完了後、コマンドプロンプト（Windows）またはターミナル（Mac）を開きます
# `node --version` を実行し、`v20.x.x` のようなバージョン番号が出ればOKです

== 初回セットアップ ==

リポジトリのフォルダに移動します。

<syntaxhighlight>
cd 自分のクローン先パス/subspaceworld
</syntaxhighlight>

> [!note] 依存パッケージは無し
> このプロジェクトは外部依存ゼロで設計されているため、`npm install` を実行する必要はありません。`node` と `npm` だけ入っていれば動きます。

== 記事データを再生成する ==

`content/` の記事ファイルを変更したら、サイト用のデータに反映する必要があります。

<syntaxhighlight>
npm run build:content
</syntaxhighlight>

これで `src/data/generated/content-data.js` が更新されます。新規記事の追加・既存記事の編集・ファイルのリネームをした際は必ず実行してください。

== ローカルサーバーを起動する ==

<syntaxhighlight>
npm run serve
</syntaxhighlight>

起動すると `http://localhost:4173/` でアクセスできるようになります。ブラウザでこのURLを開いてください。

> [!tip] 終了方法
> サーバーを止めるには、コマンドプロンプト/ターミナル上で '''Ctrl+C''' を押します。

== 確認の流れ ==

# 記事を編集する
# `npm run build:content` で再生成する
# ブラウザを再読み込み（'''Ctrl+R''' / '''Cmd+R'''）する
# 表示を確認する
# 必要に応じて編集に戻る

== よくある質問 ==

=== 表示が崩れている ===

[[はじめての記事投稿ガイド]]の「記事の表示がおかしい」を参照してください。記事側の記法を疑う前に、まずパーサー側の不具合を疑って報告するのが推奨です。ローカル環境で再現できれば、報告時にスクリーンショットや具体的な記事ソースも添えやすくなります。

=== ポートが既に使われている ===

別のアプリが4173番ポートを使っている場合、環境変数 `PORT` で別ポートを指定できます。

<syntaxhighlight>
PORT=5000 npm run serve
</syntaxhighlight>

Windowsのコマンドプロンプトでは以下のように書きます。

<syntaxhighlight>
set PORT=5000 && npm run serve
</syntaxhighlight>

=== build:content と build:site の違い ===

{| class="wikitable"
! コマンド !! 用途
|-
| `npm run build:content` || 記事ファイル → JSデータの変換のみ（ローカル確認はこちらでOK）
|-
| `npm run build:site` || 上記＋HTML出力。GitHub Pagesへのデプロイ用で、ローカルでは通常不要
|-
| `npm run serve` || ローカルサーバー起動（`npm run dev` も同じ）
|-
| `npm test` || 自動テストの実行
|}

=== 変更したのにブラウザに反映されない ===

* `npm run build:content` を実行し忘れていないか確認してください
* ブラウザのキャッシュが残っている場合があります。'''Ctrl+Shift+R'''（Windows）または '''Cmd+Shift+R'''（Mac）でハードリロードしてみてください
