**はじめての記事投稿ガイド**は、GitHubアカウントを持っておらず、Gitに触ったこともない方を対象に、subspaceworldへ記事を投稿するまでの全手順を説明する記事です。

== 投稿までの全体像 ==

このプロジェクトでは、記事を書いてから公開されるまでに以下の流れを踏みます。

# 記事を書く
# 変更を記録する（コミット）
# GitHubにアップロードする（プッシュ）
# レビューを依頼する（プルリクエスト）
# 管理者が承認し公開する（マージ）

難しそうに見えますが、一つ一つは単純な操作です。このガイドではGUIツール（GitHub Desktop）を使うので、コマンドを打つ必要はありません。

== GitHubアカウントを作る ==

=== サインアップ ===

# ブラウザで '''https://github.com''' を開きます
# 画面右上の '''「Sign up」''' をクリックします
# 以下を入力します
#* '''Email''' — 普段使っているメールアドレス
#* '''Password''' — 15文字以上、または数字と小文字を含む8文字以上
#* '''Username''' — 好きな英数字のユーザー名（あとから変更も可能です）
# メール認証を求められるので、届いた確認メールのリンクをクリックします
# プランの選択画面が出たら '''「Free」''' のままで問題ありません

=== プロフィールの設定（任意） ===

アカウント作成後、右上のアイコン → '''Settings''' → '''Profile''' からプロフィールを編集できます。記事投稿には必須ではありません。

== GitHub Desktopをインストールする ==

GitHub Desktopは、Gitの操作をボタンで行えるアプリです。コマンドは一切不要です。

# '''https://desktop.github.com''' にアクセスします
# '''「Download for Windows」'''（または macOS）をクリックします
# ダウンロードしたインストーラーを実行します
# 起動後 '''「Sign in to GitHub.com」''' をクリックし、先ほど作ったGitHubアカウントでログインします
# 名前とメールアドレスの設定画面が出たら、そのまま '''「Finish」''' で完了です

== リポジトリをクローンする（PCにコピーする） ==

「クローン」とは、GitHub上のプロジェクトをまるごと自分のPCにコピーすることです。このリポジトリは公開リポジトリなので、GitHubアカウントさえあれば誰でもクローンできます。

# GitHub Desktopを開きます
# メニューの '''File → Clone repository''' をクリックします
# '''「URL」''' タブを選択します
# URLの欄に `https://github.com/asteain-ninia/subspaceworld` を貼り付けます
# '''「Local path」''' で保存先フォルダを確認します（デフォルトのままでもOKです）
# '''「Clone」''' をクリックします
# ダウンロードが完了すると、GitHub Desktopにリポジトリが表示されます

> [!tip] ポイント
> クローンは最初の1回だけ行います。次回以降はGitHub Desktopを開けばそのまま使えます。

== プルする（最新の状態に同期する） ==

このプロジェクトは複数人で編集するため、自分がクローンした後に他の人が記事を追加・変更していることがあります。「プル」は、GitHubにある最新の変更を自分のPCに取り込む操作です。

# GitHub Desktopの上部で '''「Current branch」''' が '''「main」''' になっていることを確認します
# '''「Fetch origin」''' ボタンをクリックします（GitHubに新しい変更があるか確認します）
# 新しい変更が見つかった場合、ボタンが '''「Pull origin」''' に変わるので、それをクリックします

> [!warning] いつプルすべきか
> 新しいブランチを作って記事を書き始める'''前に'''、必ずプルして最新状態にしてください。古い状態のまま作業すると、他の人の変更と衝突（コンフリクト）する可能性があります。しばらく作業していなかった場合は特に注意しましょう。

== ブランチを作る（作業スペースを分ける） ==

「ブランチ」は、他の人の作業に影響を与えずに自分の変更を進めるための仕組みです。記事を書く前に、必ず新しいブランチを作ります。

# GitHub Desktopの上部にある '''「Current branch」''' をクリックします
# '''「New branch」''' をクリックします
# ブランチ名を入力します
# '''「Create branch」''' をクリックします

=== ブランチ名のつけ方 ===

内容がわかる短い名前を英数字でつけてください。日本語は避けましょう。

{| class="wikitable"
! 例 !! 意味
|-
| `add/my-first-article` || 新規記事の追加
|-
| `add/ete-republic` || エテ合議国の記事を追加
|-
| `fix/typo-in-deere` || デーレ共和国の誤字修正
|-
| `update/qafik-history` || ガフィークの歴史セクション更新
|}

> [!warning] 注意
> ブランチは記事を書き始める'''前に'''作ってください。mainブランチで直接作業すると、プルリクエストが作れません。mainで作業してしまった場合の対処は[[mainブランチでの誤コミットからの復旧手順]]を参照してください。

== 記事を書く ==

=== ファイルの置き場所 ===

記事ファイルは `content/` フォルダの中に置きます。クローンしたフォルダをエクスプローラーで開くと、以下のような構成になっています。

* `subspaceworld/content/` — ここに記事を置きます
* `subspaceworld/content/assets/` — 画像ファイルを置きます
* `subspaceworld/content/samples/` — サンプル記事です（ここには置かないでください）

=== 記事ファイルを作る ===

テキストエディタ（メモ帳、VS Code、Obsidianなど何でもOKです）で新しいファイルを作り、`content/` フォルダに保存します。

* '''ファイル名''' — 記事のタイトルをそのままつけます（日本語OKです）
* '''拡張子''' — `.md`（Markdown）、`.wiki`、`.txt` のいずれかです
* 例 — `エテ合議国.md`、`デーレ共和国.md`

=== 記事の書き方 ===

記事はMarkdown記法またはMediaWiki記法で書けます。同一ファイル内で両方の記法を混ぜても大丈夫です。

==== 最小限の記事の例 ====

<syntaxhighlight>
**エテ合議国**（エテごうぎこく）は、[[出界]]に存在する国家である。

== 概要 ==

東方で[[デーレ共和国]]と隣接する。
山がちながら初期文明の一つを創始した。

== 歴史 ==

エテ文明という最古級の文明を持つ。
</syntaxhighlight>

==== よく使う書式 ====

{| class="wikitable"
! 書きたいもの !! 書き方
|-
| 太字 || `<nowiki>**太字**</nowiki>` または `<nowiki>'''太字'''</nowiki>`
|-
| 斜体 || `<nowiki>*斜体*</nowiki>` または `<nowiki>''斜体''</nowiki>`
|-
| 内部リンク || `<nowiki>[[ページ名]]</nowiki>`
|-
| 表示名付きリンク || `<nowiki>[[ページ名|表示名]]</nowiki>`
|-
| 見出し（大） || `== 見出し ==`
|-
| 見出し（中） || `=== 見出し ===`
|-
| 箇条書き || `* 項目`
|-
| 番号付きリスト || `# 項目`
|}

==== 画像を使う場合 ====

# 画像ファイルを `content/assets/` フォルダにコピーします
# 記事の中で `<nowiki>![[画像ファイル名.png]]</nowiki>` と書きます

==== frontmatter（省略可） ====

記事の先頭に以下のような情報ブロックを書くと、タグや要約などを設定できます。必須ではありませんが、書いておくと管理者の手間が減ります。

<syntaxhighlight>
---
title: エテ合議国
tags:
  - world
  - country
draft: false
authors:
  - あなたのユーザー名
summary: エテ合議国についての概要記事
---
</syntaxhighlight>

> [!tip] ヒント
> `draft: true` にすると下書き扱いになり、公開サイトには反映されません。書きかけの記事にはこれを指定しておくと安全です。

== コミットする（変更を記録する） ==

「コミット」は、自分の変更内容に名前をつけて記録する操作です。

# GitHub Desktopに戻ると、左側に変更されたファイルの一覧が表示されています
# 変更内容を確認します（右側に差分が表示されます）
# 左下の '''「Summary」''' 欄にコミットメッセージを入力します
# '''「Commit to （ブランチ名）」''' ボタンをクリックします

=== コミットメッセージの書き方 ===

何を変更したかがわかる短い文を書きます。日本語でOKです。

{| class="wikitable"
! 良い例 !! 悪い例
|-
| `エテ合議国の記事を追加` || `更新`
|-
| `デーレ共和国に歴史セクションを追記` || `変更しました`
|-
| `ガフィークの誤字を修正` || `修正`
|}

> [!tip] ポイント
> コミットは何回でもできます。ひとまとまりの作業ごとにコミットしておくと、あとから変更履歴を追いやすくなります。

== プッシュする（GitHubにアップロードする） ==

「プッシュ」は、自分のPC上のコミット記録をGitHubにアップロードする操作です。

# GitHub Desktop上部の '''「Publish branch」''' ボタン（初回）、または '''「Push origin」''' ボタン（2回目以降）をクリックします
# アップロードが完了するまで少し待ちます

これで変更がGitHub上に反映されます。

> [!note] 補足
> プッシュするには、リポジトリへの書き込み権限が必要です。公開リポジトリでもプッシュには管理者からのCollaborator招待が必要ですので、まだの方は管理者に連絡してください。招待メールが届いたら '''「View invitation」''' をクリックして承諾します。

== プルリクエストを作る（レビューを依頼する） ==

「プルリクエスト（PR）」は、自分の変更を管理者にレビューしてもらい、公開に組み込んでもらうための依頼です。

=== GitHub Desktopから作る場合 ===

# プッシュ後に表示される '''「Create Pull Request」''' ボタンをクリックします
# ブラウザでGitHubのPR作成画面が開きます

=== ブラウザから作る場合 ===

# ブラウザでリポジトリのページ（`https://github.com/asteain-ninia/subspaceworld`）を開きます
# 黄色い帯で '''「Compare & pull request」''' と出ていたらそれをクリックします
# 出ていなければ '''「Pull requests」''' タブ → '''「New pull request」''' をクリックし、自分のブランチを選びます

=== PRの内容を書く ===

{| class="wikitable"
! 項目 !! 書くこと
|-
| '''Title''' || 変更の要約（例: `エテ合議国の記事を追加`）
|-
| '''Description''' || 補足説明があれば自由に書いてください
|}

記入したら '''「Create pull request」''' をクリックして完了です。

> [!tip] これで作業は完了です
> あとは管理者がレビューしてくれるのを待ちましょう。

== レビュー後の流れ ==

=== 修正を求められた場合 ===

管理者からコメントで修正を求められることがあります。その場合は以下の手順で対応します。

# GitHub Desktopで同じブランチが選択されていることを確認します
# 指摘箇所をテキストエディタで修正します
# 再度コミットします（「コミットする」の手順と同じです）
# 再度プッシュします（「プッシュする」の手順と同じです）

プルリクエストに自動的に修正が反映されます。新しいPRを作り直す必要はありません。

=== 承認された場合 ===

管理者がプルリクエストを承認し「マージ」すると、記事が公開サイトに反映されます。反映には数分かかることがあります。

=== 次の記事を書くとき ===

# GitHub Desktopで '''「Current branch」''' → '''「main」''' に切り替えます
# 「プルする」の手順に従って最新状態に同期します
# 新しいブランチを作り、次の記事に取りかかります

== 用語集 ==

; リポジトリ
: プロジェクト全体の保管庫です。ファイル・履歴・設定がすべて入っています。
; クローン
: リポジトリをGitHubからPCにまるごとコピーすることです。
; ブランチ
: 他の作業に影響を与えない独立した作業スペースです。木の枝（branch）のイメージです。
; main
: 公開版のブランチです。ここに直接変更を加えないでください。
; コミット
: 変更をひとまとめにして記録することです。セーブポイントのようなものです。
; プッシュ
: PCに記録したコミットをGitHubにアップロードすることです。
; プルリクエスト（PR）
: 「自分の変更をmainに取り込んでほしい」という依頼です。レビューの単位になります。
; マージ
: PRの内容をmainに統合することです。管理者が行います。
; プル
: GitHubにある最新の変更を自分のPCに取り込むことです。フェッチ＋変更の反映をまとめて行います。
; フェッチ
: GitHubに新しい変更があるかを確認することです。確認だけで、まだ自分のPCには反映されません。
; 差分（diff）
: 変更前と変更後の違いのことです。
; frontmatter
: 記事ファイルの先頭に書くメタデータ（タイトル・タグなど）のことです。

== よくある質問 ==

=== コミットメッセージを間違えた ===

そのまま進めて問題ありません。PRのタイトルや説明文で正しく伝えれば大丈夫です。

=== ファイル名を間違えた ===

そのまま正しい名前にリネームして、もう一度コミットすれば修正されます。

=== 画像がうまく表示されない ===

* 画像ファイルが `content/assets/` に置かれているか確認してください
* ファイル名が記事内の記述と一致しているか確認してください（大文字小文字も区別されます）

=== 記事の表示がおかしい ===

記事を書いて公開されたあと、表示が崩れたり意図しない見た目になっている場合は、サイト側のパーサーが対応していない記法を使っている可能性があります。管理者に連絡してください。

=== Gitやコマンドラインに慣れてきたら ===

GitHub Desktopの代わりにコマンドラインでGit操作することもできます。操作は同じ流れ（クローン→ブランチ作成→編集→コミット→プッシュ→PR）です。ただし、GitHub Desktopで問題なく使えている場合は無理に切り替える必要はありません。
