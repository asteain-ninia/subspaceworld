**mainブランチでの誤コミットからの復旧手順**は、[[はじめての記事投稿ガイド]]の補足記事です。ブランチを作り忘れてmainで作業・コミットしてしまった場合の対処方法をまとめます。

== まだコミットしていない場合 ==

変更したファイルはあるが、まだコミットボタンを押していない状態であれば簡単に対処できます。

# GitHub Desktopの上部にある '''「Current branch」''' をクリックします
# '''「New branch」''' をクリックします
# ブランチ名を入力して '''「Create branch」''' をクリックします

これだけで、未コミットの変更はそのまま新しいブランチに引き継がれます。あとは通常通りコミット・プッシュ・プルリクエストの手順に進んでください。

== コミットしたが、まだプッシュしていない場合 ==

mainブランチでコミットしてしまったが、まだ '''「Push origin」''' を押していない状態であれば、手元で修正できます。GitHub Desktopで以下の手順を行ってください。

=== 手順1: コミットを新しいブランチに移す ===

# '''「Current branch」''' をクリックし、'''「New branch」''' をクリックします
# ブランチ名を入力して '''「Create branch」''' をクリックします
# 「mainの現在のコミットからブランチを作りますか？」のような確認が出たら、そのまま作成します

これで、mainで行ったコミットが含まれた新しいブランチができます。

=== 手順2: mainブランチを元に戻す ===

# '''「Current branch」''' をクリックし、'''「main」''' に切り替えます
# メニューの '''「Branch → Undo last commit...」''' をクリックします（コミットが複数ある場合は、元に戻したい回数だけ繰り返します）
# mainが元の状態に戻ったことを確認します（変更ファイルがなくなっていればOKです）

> [!warning] 注意
> 手順2で「Undo last commit」すると、コミットの内容が未コミットの変更として残ります。この変更は不要なので、'''「Repository → Discard all changes...」''' で破棄してください。手順1で作ったブランチにはコミット済みの変更がすでに入っているので、失われることはありません。

=== 手順3: 新しいブランチから作業を再開する ===

# '''「Current branch」''' をクリックし、手順1で作ったブランチに切り替えます
# ここからは通常通りプッシュ・プルリクエストの手順に進みます

== プッシュしてしまった場合 ==

'''「Push origin」''' を押してGitHubにアップロード済みの場合は、'''管理者に相談してください'''。GitHubのmainブランチに直接変更が反映されてしまっているため、リポジトリ側での対応が必要になります。

管理者へ連絡する際には、以下を伝えるとスムーズです。

* 何をコミットしたか（追加・変更したファイル名）
* コミットした回数（1回か複数回か）

== 予防策 ==

* 作業を始める前に、GitHub Desktop上部の '''「Current branch」''' 表示を確認する習慣をつけましょう
* mainと表示されていたら、必ず新しいブランチを作ってから編集を始めてください
